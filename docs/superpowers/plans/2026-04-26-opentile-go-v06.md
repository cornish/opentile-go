> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add OME TIFF support as the fourth format opentile-go handles. Two parity references (opentile-py + tifffile) make the multi-image deviation testable without baking in upstream's last-wins drop.

```go
const (
    // ...existing...
    TagSubIFDs uint16 = 330
)

// SubIFDOffsets returns the SubIFDs tag (330) value as a slice of uint64
// offsets, or (nil, false) if absent. Used by OME TIFF where lower-resolution
// pyramid levels live in SubIFDs of the base page rather than as top-level
// IFDs.
func (p *Page) SubIFDOffsets() ([]uint64, bool) {
    if !p.HasTag(TagSubIFDs) {
        return nil, false
    }
    offs, err := p.arrayU64(TagSubIFDs)
    if err != nil {
        return nil, false
    }
    return offs, true
}
```

```go
// PageAtOffset reads a single IFD at offset off and returns it as a Page.
// Used by callers that need to follow SubIFD pointers (TIFF tag 330) — the
// SubIFD chain is not part of the top-level next-IFD walk.
//
// Does not advance any walker; returns one Page per call. Caller validates
// the resulting Page (check ImageWidth/ImageLength etc.).
func (f *File) PageAtOffset(off uint64) (*Page, error) {
    ifd, err := readIFD(f.br, off, f.bigtiff, f.allowedFlags)  // signature TBD; mirror existing IFD reader
    if err != nil {
        return nil, fmt.Errorf("tiff: read IFD at %d: %w", off, err)
    }
    return newPage(ifd, f.br), nil
}
```

```python
def serve(path):

    tf = tifffile.TiffFile(path)
```

```markdown
## Deviations from upstream Python opentile

Behaviours where opentile-go intentionally differs from upstream
Python opentile 0.20.0. Each entry names the upstream behaviour, our
deviation, and why.

- **NDPI synthesised label** (since v0.2). Upstream's `NdpiTiler.labels`
  returns empty; we synthesise a label by cropping the left 30% of
  the overview page. Opt-out via `opentile.WithNDPISynthesizedLabel(false)`.
  Reason: Python opentile chose not to expose NDPI labels at all;
  Aperio-style label affordance is more useful for downstream
  consumers. Original L14 entry: §2.

- **NDPI Map page surfacing** (since v0.4). Upstream Python opentile
  filters out tifffile's `series.name == 'Map'` pages; we expose
  them as `Tiler.Associated()` entries with `Kind() == "map"`.
  Reason: tifffile already classifies them; surfacing matches what
  the underlying TIFF carries. Original R13 entry: §1.

- **Multi-image OME exposure** (since v0.6). Upstream's base Tiler
  loop assumes one main pyramid per file; in multi-image OME files
  (e.g., Leica-2.ome.tiff with 4 main pyramids) it silently drops
  all but the last via an unintentional last-wins assignment. We
  expose all main pyramids via the new `Tiler.Images()` API, with
  `Tiler.Levels()` continuing to refer to `Images()[0]` for
  backwards compatibility. Reason: encoding upstream's accidental
  drop in our port would bake in an upstream oversight.
```
