## Task 5: `internal/jpegturbo.FillFrame` cgo entry point

**Goal:** Add a fourth public function to `internal/jpegturbo` that produces a solid-colour version of an input JPEG (via tjTransform with a CUSTOMFILTER overwriting all DCT coefficients).

**Files:**
- Modify: `internal/jpegturbo/turbo.go` (signature + doc)
- Modify: `internal/jpegturbo/turbo_cgo.go` (cgo implementation)
- Modify: `internal/jpegturbo/turbo_nocgo.go` (stub)
- Modify: `internal/jpegturbo/turbo_cgo_test.go` (unit + determinism tests)

- [ ] **Step 0: Confirm upstream**

`opentile/jpeg/jpeg_filler.py:173-280` is the upstream class. Key
mechanics:
1. `tjInitTransform` returns a transform handle.
2. CUSTOMFILTER callback: overwrite every DCT coefficient with a
   constant — DC = `LuminanceToDCCoefficient(luminance)`, AC = 0.
3. Pass `TJXOPT_PERFECT | TJXOPT_CROP` flags (same as our other
   transform calls).
4. `tjFree` the output buffer; copy to a Go-managed slice.

The CUSTOMFILTER callback is the only material difference from our
existing `CropWithBackground` callback — there, only OOB blocks get
overwritten; here, every block does.

// derive a "blank" replacement tile from a known-valid frame.

// Concurrency: safe for parallel use. Per-call tjInitTransform /

Key behaviour:
- `_create_blank_tile`: walks `databytecounts` for the first non-zero
  index, reads that frame, splices JPEGTables (no APP14 — Philips
